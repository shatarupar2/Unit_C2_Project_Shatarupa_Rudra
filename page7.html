File Path & Enhaced Text Editors

File Paths

What is a file path?

A file path is the written route a computer will use to locate a file.

Different types of file paths

The absolute file path uses the full path, starting from the root of the file. It states the full location of the file from beginning to end, no matter where you are in the system

An example of this would be: 
            /home/ILOVECODE/projects/index.html

The relative file path uses the currect file being worked on in order to locate the file without much effort. It references files that are located within the same directory or subdirectory of the current folder, without stating the full path of it.

An example of this would be:
            ./index.html

Enhanced Text Editors

What are Enhanced Text Editors?

Developers use enhanced text editors as a way to check and help write code.

Different Text Editors for Different Systems

Windows

Windows user mostly use Notepad++ as their enhanced text editor.

Mac

Mac users mostly use BBEDIT as their enhanced text editor.

Linux

Linux users mostly use EMACS as their enhanced text editors.

Integrated Development Environments

Developers also use Integrated Development Environments, a program that guides them in numerous ways. It offers support with code editors and other tools to help execute the code.

Validator

A validator is used to ensure code is accurate through the lens of specific standards. It helps catch errors and maintain compatibility across systems.
